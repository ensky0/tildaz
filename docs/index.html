Release Notes

What It Optimizes

Fast access, native behavior, no terminal compromise.

Hotkey

Drop down from anywhere

One global hotkey shows or hides the terminal. The window docks to the screen with your cursor and supports top, bottom, left, and right layouts.

Tabs

Multiple sessions, one surface

Open, close, switch, drag reorder, and rename tabs. Rename behavior is IME-aware, including Hanja candidate replacement, and commits predictably on focus loss.

Rendering

DirectWrite and CoreText

Windows uses DirectWrite with Direct3D 11. macOS uses CoreText with Metal. Both paths use native font systems and GPU atlases.

Unicode

Hangul, CJK, emoji, IME

Inline pre-edit, wide characters, color emoji, ZWJ clusters, and Hanja / kanji / hanzi candidate positioning and replacement are part of the core path.

PTY

Real shell integration

Bundled OpenConsole ConPTY on Windows; POSIX PTY with login_tty on macOS. Child shells receive consistent terminal env.

Config

One schema across platforms

The JSON config is strict, shared, and explicit about units. Font fallback supports up to 8 families including the primary font.

Windows

ConPTY + Direct3D 11

Ships with Microsoft OpenConsole runtime files to reduce system ConPTY variance, while falling back to the OS implementation when needed.

  • DirectWrite ClearType glyph rasterization
  • Bundled OpenConsole.exe and conpty.dll
  • Win32 global hotkey and native window behavior

macOS

POSIX PTY + Metal

Uses AppKit, CoreText, and Metal directly, with NSTextInputClient support for IME composition, candidate anchoring, and Hanja reconversion.

  • Universal Apple Silicon + Intel release artifact
  • CoreText glyph fallback and Metal atlas rendering
  • Input Monitoring + Accessibility for global control
